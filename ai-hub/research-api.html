🔬 Research Findings API

Public REST API exposing Wyltek's AI research findings. Zero auth for consumers — designed for agent-to-agent access.

Base URL https://api.wyltekindustries.com
Live

Quick Reference

GET /api/research/findings List all findings GET /api/research/findings?tag=<tag>&meta=1&limit=N Filtered list with metadata GET /api/research/findings/:id Full finding content GET /api/research/findings/:id?format=markdown Raw markdown GET /api/research/tags All tags with counts GET /api/research/health Health check
🔓 No authentication required All endpoints are public and CORS-enabled. Any agent, script, or browser can call them directly. Rate limits apply at Cloudflare edge level.

Query Parameters

Parameter Type Description
tagoptional string Filter by tag (case-insensitive). Requires meta=1.
metaoptional 0 | 1 Include parsed metadata (title, date, tags, summary) per finding. Slower — fetches each file.
limitoptional integer Maximum number of results to return. Default: all.

🤖 Agentic Usage

This API is designed for agent-to-agent consumption. No API keys, no sessions, no webhooks — just plain HTTP GET. Agents can pull research context on demand and inject it into their working memory or LLM context window.

📋 Pre-task Research Injection

Before starting a task, query for relevant findings and inject them as context. Useful for OpenClaw agents, LangChain tools, or any agent framework.

  • Fetch findings tagged fiber before working on payment channel tasks
  • Pull the latest ckb findings to inform transaction-building decisions
  • Use meta=1&limit=5 for a fast 5-finding summary, then fetch full content on demand
# In an OpenClaw skill or heartbeat task
BASE = "https://api.wyltekindustries.com"

async def get_research_context(tag, limit=5):
    r = requests.get(f"{BASE}/api/research/findings",
                     params={"meta": 1, "tag": tag, "limit": limit})
    findings = r.json()["findings"]
    # Build context block for LLM
    context = "\n\n".join(
        f"## {f['title']}\n{f.get('summary', '')}"
        for f in findings
    )
    return context
import requests

def fetch_research_context(tag: str, limit: int = 5) -> str:
    """Pull relevant research and format for LLM injection."""
    BASE = "https://api.wyltekindustries.com"
    r = requests.get(
        f"{BASE}/api/research/findings",
        params={"meta": "1", "tag": tag, "limit": str(limit)}
    )
    findings = r.json().get("findings", [])
    return "\n\n---\n\n".join(
        f"**{f['title']}** ({f.get('date','?')})\n{f.get('summary','')}"
        for f in findings
    )

# Example: inject into LLM prompt
context = fetch_research_context("fiber", limit=3)
prompt = f"Use this research context:\n\n{context}\n\nNow answer: ..."

🔍 On-Demand Deep Fetch

List findings first (lightweight), then fetch full content only for the most relevant ones. Minimises token burn on context injection.

  • Get the ID list first — cheap, just filenames and sizes
  • Score/rank IDs against your query using your own logic
  • Fetch ?format=markdown for the top N hits only
Pattern
import requests, re

BASE = "https://api.wyltekindustries.com"

def smart_fetch(query_keywords: list[str], top_n=3) -> list[str]:
    # 1. Get cheap listing
    ids = [f["id"] for f in
           requests.get(f"{BASE}/api/research/findings").json()["findings"]]

    # 2. Score by keyword match in ID string
    scored = sorted(
        ids,
        key=lambda id: sum(kw.lower() in id for kw in query_keywords),
        reverse=True
    )

    # 3. Fetch full markdown for top hits only
    results = []
    for finding_id in scored[:top_n]:
        md = requests.get(
            f"{BASE}/api/research/findings/{finding_id}",
            params={"format": "markdown"}
        ).text
        results.append(md)
    return results

🗺️ Discovery — What's Been Researched?

Use the tags endpoint at startup to discover what topics are covered before planning research tasks. Avoids duplicating work that's already been done.

  • Cache the tag list at agent boot — it's slow but only needed once
  • Check if a topic has coverage before spawning a new research crawl
  • Cross-reference with your own task queue to find gaps
import requests

BASE = "https://api.wyltekindustries.com"

# Fetch tag coverage once, cache locally
tags_data = requests.get(f"{BASE}/api/research/tags").json()
covered_tags = {t["tag"] for t in tags_data["tags"]}

def already_researched(topic: str) -> bool:
    keywords = topic.lower().split()
    return any(kw in covered_tags for kw in keywords)

# Before queuing a new research task:
if not already_researched("fiber htlc"):
    queue_research_task("fiber htlc routing")
💡 Recommended agent pattern Boot: fetch tags → cache. Per-task: query ?meta=1&tag=X&limit=5 for summaries → inject top 2–3 into LLM context. On demand: fetch individual findings with ?format=markdown when full depth is needed. This keeps token cost minimal while giving your agent strong domain context.
