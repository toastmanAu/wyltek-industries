📖 Retrieval-Augmented Generation (RAG)

What is RAG?

RAG combines knowledge retrieval + LLM generation:

  1. User asks question
  2. System retrieves relevant documents (vector search)
  3. LLM reads documents + answers based on context
  4. Result: accurate, sourced, up-to-date answers

Why RAG?

  • ✅ Reduces hallucinations (grounded in real data)
  • ✅ Keeps knowledge fresh (add new docs anytime)
  • ✅ Cheaper than fine-tuning (no retraining)
  • ✅ Citable (sources included in response)

Wyltek RAG System Architecture

Documents (research findings, blog posts)
    ↓
Chunking (300-token chunks with overlap)
    ↓
Embedding (sentence-transformers: all-MiniLM-L6-v2)
    ↓
Vector Store (FAISS: fast similarity search)
    ↓
Retrieval (top-5 similar chunks)
    ↓
LLM Generation (context-aware answering)
    ↓
Cited Response

Deploy Wyltek RAG

# Clone from GitHub
git clone https://github.com/toastmanAu/rag-system.git ~/rag-system

# Setup
cd ~/rag-system
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Run
python3 app.py 9990

# Test
curl http://localhost:9990/health
# → {"status": "ok", "service": "rag"}

Ingest Documents

curl -X POST http://localhost:9990/rag/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/doc",
    "html": "...",
    "tags": ["research", "fiber"]
  }'

Query with RAG

# Retrieve context
curl -X POST http://localhost:9990/rag/retrieve \
  -H "Content-Type: application/json" \
  -d '{
    "query": "How do I set up Fiber payments?",
    "agent_id": "kernel"
  }'

# Ask with LLM
curl -X POST http://localhost:9990/rag/ask \
  -H "Content-Type: application/json" \
  -d '{
    "query": "How do I set up Fiber payments?",
    "agent_id": "kernel",
    "use_llm": true
  }'

Multi-Agent Profiles

Configure different agents to interpret same knowledge differently:

agents.yaml:

agents:
  kernel:
    agent_name: "Kernel"
    retrieval_config:
      k_retrieve: 20
      k_rerank: 5
    source_weights:
      docs: 1.0
      research-findings: 0.8

  shannon:
    agent_name: "Shannon"
    retrieval_config:
      k_retrieve: 20
      k_rerank: 5
    source_weights:
      blog: 1.0
      research-findings: 0.6
