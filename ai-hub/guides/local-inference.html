🧠 Local LLM Inference Setup

Why Local Inference?

  • 💰 60-80% cost reduction vs API calls
  • 🔒 Keep data private (on-device)
  • ⚡ Sub-second latency (local network)
  • 🚀 Run agents 24/7 affordably

Model Selection by Hardware

Hardware Recommended Model Speed Quality
Pi 5 (8GB) qwen2.5:3b-q4 ~3 tok/s Good
N100 (15GB) qwen2.5:7b-q4 ~5 tok/s Better
x86 + GPU (RTX 3060) qwen3.5:27b ~30 tok/s Excellent
OPi5+ (16GB) qwen2.5:7b-q4 ~4 tok/s Better

Installation & Running

# Install Ollama (all platforms)
curl https://ollama.ai/install.sh | sh
ollama serve

# In another terminal:
# Pull model (one-time download)
ollama pull qwen2.5:7b

# Test
ollama run qwen2.5:7b "What is CKB?"

Quantization: Speed vs Quality

  • Q8 (8-bit): Best quality, ~2x slower, ~2x larger
  • Q4 (4-bit): Good quality, fast, recommended baseline
  • Q2 (2-bit): Very fast, lower quality, for simple tasks
  • 1.58-bit (BitNet): Experimental, ~3x faster, specialized training

Cost Analysis

For 1M tokens/month:

  • Claude API: ~$10-15
  • Local Ollama (electricity): ~$1-2
  • Savings: 85-90%

Production Setup

# Systemd service (Linux)
cat > /etc/systemd/user/ollama.service << 'UNIT'
[Unit]
Description=Ollama Service
After=network.target

[Service]
ExecStart=/usr/bin/ollama serve
Restart=on-failure
RestartSec=10

[Install]
WantedBy=default.target
UNIT

systemctl --user enable ollama
systemctl --user start ollama
