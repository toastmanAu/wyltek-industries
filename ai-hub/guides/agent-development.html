Concise when needed, thorough when it matters.

Multi-Agent Coordination

# In agent code:
const subagents = require('@openclaw/subagents');

// Spawn worker agent
const result = await subagents.spawn({
  agentId: 'worker-id',
  task: 'Research Fiber protocol',
  model: 'local-qwen:7b'
});

// Wait for result
console.log(result.output);

Example: Research Agent Loop

1. Read queue.md for research tasks
2. Pick PENDING task (HIGH > MEDIUM > LOW)
3. Fetch seed URLs
4. Call Gemini with ground truth context
5. Write findings to research/findings/
6. Ingest to RAG system
7. Auto-generate follow-up tasks
8. Loop (new wave of research)

Best Practices

  • ✅ Use RAG for grounded answers
  • ✅ Cache frequently-accessed data locally
  • ✅ Log decisions (for audit trail)
  • ✅ Use local inference when possible
  • ✅ Batch API calls (reduce latency)
  • ❌ Don't hallucinate (say "I don't know")
  • ❌ Don't spam external APIs
