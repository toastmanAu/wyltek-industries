CKB Community Fund DAO · 2024

Got one of my CKB nodes?
Here's what's new for it.

In early 2024 I built and shipped 30 Orange Pi 3B CKB full nodes to Nervos DAO voters around the world — 4W, WiFi + LAN, with a live display. This script brings them fully up to date: fixing stability bugs that shipped with the original install, adding auto-updates, a live dashboard, and ckbnode.local access from any device on your network.

SSH into your node, then run:
curl -fsSL https://raw.githubusercontent.com/toastmanAu/armbian-ckb-userpatches/master/scripts/fix-ckb-stability.sh | sudo bash
Takes 2–5 minutes · Safe to run multiple times · Nothing removed, only added
Your node: Orange Pi 3B · CKB v0.202.0 or earlier · Original project thread ↗

About these nodes
These nodes were built and distributed in early 2024 as part of a CKB Community Fund DAO proposal to improve Nervos network decentralisation. Each one is an Orange Pi 3B running CKB v0.202.0, with a small OLED display, WiFi + Ethernet, and drawing ~4W at the wall. They were shipped to Nervos DAO voters in 30 countries in January–February 2024.

The original setup was functional but bare — no auto-updates, no dashboard, and a couple of systemd bugs that cause the node to silently restart every 5 minutes. This script fixes all of that.

What gets installed

Original config files are never modified — all changes go into drop-in files you can inspect and remove at any time.

Bug Fix 01
Watchdog Crash Fix
The original service file kills CKB every 5 minutes. It has WatchdogSec=300 set, but CKB never sends the expected keepalive pings — so systemd kills it on a fixed timer, forever. Fixed with a safe drop-in override.
Critical Fix
Bug Fix 02
Restart Limit Removed
The original config gives up restarting after 5 failures in 5 minutes. If the node crashes during a bad block or OOM event, it stays offline permanently until someone SSHs in. Fixed: retries forever.
Critical Fix
Bug Fix 03
WiFi Power-Save Off
Linux puts the network card to sleep between packets. On WiFi nodes this silently drops peer connections and stalls sync. A udev rule disables it permanently and applies it immediately without a reboot.
Bug Fix
Bug Fix 04
Auto-Start on Boot
Many manually-installed nodes don't survive a power cut — systemctl enable was never run. Fixed: CKB starts automatically on boot.
Bug Fix
New 05
Application Watchdog
Polls the CKB RPC every 5 minutes. If block height hasn't advanced in 10 minutes — or the RPC is completely unreachable — it restarts the service. Catches stuck syncs that look fine to systemd but aren't actually making progress.
New Feature
New 06
Auto-Update Checker
Checks GitHub daily for new CKB releases. Shows an interactive coloured prompt next time you SSH in. One keypress to upgrade — downloads the right binary for your architecture (x86_64 or arm64), restarts cleanly.
New Feature
New 07
Live Dashboard
Installs the CKB Node Dashboard as a persistent system service. Shows block height, peer count, sync status, epoch, and uptime in a live web interface. Restarts automatically if it ever goes down.
New Feature
New 08
ckbnode.local
Sets the machine hostname and installs avahi mDNS. Your node's dashboard becomes reachable at http://ckbnode.local from any phone, laptop, or tablet on your local network — no IP address needed.
New Feature
Quality 09
Log Rotation
Without this, ckb.log grows without limit and eventually fills the disk. Configured for daily rotation, 7 days of compressed history, safe copytruncate mode so CKB keeps running during rotation.
