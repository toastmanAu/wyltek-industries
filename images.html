‚ö° How to flash
Recommended for most users. Balena Etcher handles everything ‚Äî decompression, writing, verification. Works on Windows, macOS, and Linux.
1

Download Etcher

Get it free from etcher.balena.io. Available for Windows (.exe), macOS (.dmg), and Linux (.AppImage).

2

Download the image

Grab the .img.xz file from above. You don't need to extract it ‚Äî Etcher handles compressed images directly.

3

Flash

Open Etcher ‚Üí Flash from file ‚Üí select your image ‚Üí select your SD card ‚Üí Flash!. Takes 3‚Äì8 minutes depending on card speed. Etcher verifies after writing automatically.

4

First boot

Insert card, power on. First boot takes 60‚Äì90 seconds while it expands the filesystem. SSH is enabled by default ‚Äî ssh root@<ip> or check your router for the IP. Default password is in the release notes.

For Linux and macOS users. Faster than Etcher once you know your disk device. Be careful with the device path ‚Äî double-check before running.
1

Find your SD card device

# Linux
lsblk
# Look for your card ‚Äî e.g. /dev/sdb or /dev/mmcblk0

# macOS
diskutil list
# Look for your card ‚Äî e.g. /dev/disk2
2

Flash with progress bar

# Linux ‚Äî install pv first: sudo apt install pv
xz -dc wyltek-opi3b-ckbnode-*.img.xz | \
  pv -s $(xz -l wyltek-opi3b-ckbnode-*.img.xz | tail -1 | awk '{print $5}') | \
  sudo dd of=/dev/sdX bs=4M conv=fsync

# macOS (no pv needed, use disk not rdisk for safety check)
xz -d wyltek-opi3b-ckbnode-*.img.xz
sudo dd if=wyltek-opi3b-ckbnode-*.img of=/dev/rdisk2 bs=4m

Replace /dev/sdX / /dev/rdisk2 with your actual device. Wrong device = data loss.

3

Sync and eject

sync && sudo eject /dev/sdX
All images are SHA256 checksummed and GPG-signed. Verifying takes 30 seconds and confirms the image wasn't corrupted in transit.
1

Verify SHA256

# Linux / macOS
sha256sum -c wyltek-opi3b-ckbnode-*.img.xz.sha256

# Windows (PowerShell)
Get-FileHash wyltek-opi3b-ckbnode-*.img.xz -Algorithm SHA256

Should print OK. If it fails, the download is corrupted ‚Äî try again.

2

Verify GPG signature (optional, paranoid mode)

# Import Wyltek signing key
gpg --keyserver keys.openpgp.org --recv-keys CFF9573B87BB90B7

# Verify
gpg --verify wyltek-opi3b-ckbnode-*.img.xz.asc

Should show Good signature from "Wyltek Industries".
