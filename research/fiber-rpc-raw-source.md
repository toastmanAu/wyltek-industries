## Summary

This research note details the JSON field names, types, and serialization formats for key Fiber RPC methods: `open_channel`, `send_payment`, `new_invoice`, and `list_channels`. The information is extracted directly from the Rust source code of the `nervosnetwork/fiber` repository, specifically from the `crates/fiber-lib/src/rpc` module. These definitions are crucial for the FiberQuest Node.js sidecar to correctly interact with the Fiber node for managing payment channels and sending micropayments. Serialization helpers like `U128Hex`, `U64Hex`, `DisplayFromStr`, and `EntityHex` are used to represent various data types as hex strings or display strings in the JSON interface.
